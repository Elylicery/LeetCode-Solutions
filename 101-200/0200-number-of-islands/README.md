# LeetCode 200. 岛屿数量

## 思路一: 并查集

**核心思想：**
1. 将每个 '1' 视为一个独立节点
2. 遍历网格，对每个 '1'，检查其**右邻居**和**下邻居**（避免重复）
3. 如果邻居也是 '1'，则 **union** 当前节点和邻居
4. 最终**连通分量数量** = 岛屿数量

>✅ 为什么只检查右和下？
因为左和上已经在之前的遍历中处理过，避免重复 union。

**解题步骤：**
首先：二维转一维映射，构建并查集

并查集底层是一维数组，需将二维坐标 (i, j) 映射为一维索引：
`const index = i * n + j; // n = grid[0].length`

其次，并查集Union，合并岛屿
`if (uf.union(current, right)) {landCount--; }`

复杂度:
* 时间复杂度 O(m × n × α(mn)) ≈ O(mn)
* 空间复杂度 O(mn)（并查集数组）
